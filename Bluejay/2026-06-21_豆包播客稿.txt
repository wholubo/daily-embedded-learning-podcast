今天我们来聊一个非常酷的嵌入式开源项目，叫做 Bluejay。它不是咱们平时用的那种蓝鸟社交软件，而是一套专门给多旋翼无人机用的数字电调固件。电调，就是电子调速器，它的工作就是接收飞控发来的信号，然后精确控制无刷电机的转速。Bluejay 运行在 Silicon Labs 的 EFM8BB 系列微控制器上，这个芯片只有 256 个字节的 RAM 和 8KB 的 Flash，却要完成微秒级的 PWM 输出、实时解码数字协议、检测电机反电动势并完成换相，还要把转速和温度回传给飞控。在这么小的芯片上实现这么多功能，项目是用 8051 汇编语言写的，每一行代码都直接对应到硬件寄存器的操作。

Bluejay 在 GitHub 上有 552 个星，目前社区非常活跃，有配套的图形化配置工具，还有 Discord 社群。它基于经典的 BLHeli_S 固件做了大量改进，比如支持 DShot 150、300 和 600 三种速率的数字协议，双向 DShot 遥测，可选的 PWM 频率从 24 千赫兹一直到 96 千赫兹，还有 PWM 抖动技术来提升油门分辨率。对从软件工程师转型嵌入式的人来说，Bluejay 是一个极好的学习案例。因为它把 RTOS 级别的实时调度、通信协议栈、外设驱动和实时控制全部压缩在一个 8 位单片机里，如果你能理解它的设计思路，再去看 Zephyr 或者 Embassy 这类现代嵌入式框架，就会有更深刻的理解。

为什么这个项目值得你花时间研究呢？先说活跃度，项目一直在更新，有定期的 release，有配套的配置器，社区反响很好。其次它的工程完整度很高，从汇编源码到 Makefile 构建系统，它通过 Wine 调用 Keil C51 工具链来编译，还提供了批量编译脚本，能生成上百种不同硬件组合的固件文件。烧录工具有 efm8load.py，用户端有 Bluejay Configurator 图形界面。这是一条完整的从源码到产品的交付链路。

从嵌入式学习的角度说，Bluejay 要解决的工程问题是：在资源极度受限的 8 位 MCU 上，用最高精度控制一个物理旋转的电机。这跟普通后端开发面对的完全是另一个维度的问题。后端工程师习惯了几 GB 的磁盘和数 GB 的内存，而 Bluejay 只有 256 字节的 RAM 和 8KB 的 Flash，连 C 语言运行时的堆栈开销都承受不起，所以只能用汇编。后端对毫秒级的 API 响应时间觉得满意，这个电调的 PWM 周期却是微秒级的，错过了一个节拍，电机就可能失步，导致无人机炸机。后端有操作系统线程调度，这里只有六个硬件中断优先级，每个中断的上下文切换全靠程序员手工管理。调试手段也很不同，后端有 GDB 或者 Sentry，这里靠的是 LED 闪烁和电机发出的蜂鸣声，简单但有效。

我们来看看 Bluejay 的技术架构。它运行在 Silicon Labs 的 EFM8BB 系列芯片上，这个芯片内部有四个定时器、一个可编程计数器阵列 PCA、一个模数转换器、一个模拟比较器，以及 UART 串口。定时器各自分工明确。Timer0 负责生成 DShot 遥测信号，Timer1 负责解码 DShot 帧，Timer2 用于换相计时和 RPM 计算，Timer3 处理过零检测超时和提前角计时。PCA 被配置成硬件 PWM 输出，不需要 CPU 干预，而 Int0 和 Int1 中断负责捕获 DShot 信号的边沿。整个系统是中断驱动的，CPU 大部分时间在等待，只在关键决策点执行运算。

DShot 解码是 Bluejay 的一个核心模块，我们可以仔细看看。DShot 是一种数字串行协议，每帧十六位，其中十二位是油门值，四位是校验和。它使用类似曼彻斯特的编码方式——用高电平的宽度来区分零和一。Timer1 中断触发后，解码器先检查帧的长度是否在有效窗口内，这样可以防止噪声误触发。然后逐位解码，做 XOR 校验和验证，把反码数据还原，减去 96 的偏移，得到 0 到 2047 之间的油门值。如果收到的数值小于 96，它会被识别为 DShot 命令，比如进入编程模式或者读取配置参数。

无刷电机的控制部分更精彩。Bluejay 采用无传感器方案，通过检测反电动势的过零点来判断换相时机。流程分为三个阶段：启动阶段是强制换相，逐步增加功率让电机转起来，这时候还没有反电动势信号可用；接着是初始运行阶段，这时候反电动势信号开始出现，但还不够稳定；最后进入同步运行阶段，反电动势过零检测驱动换相循环。电机每转一圈需要六步换相，所以代码里有大量计时器参数来控制提前角、退磁检测和功率限制。如果检测到退磁严重，系统会自动降低功率，防止电机失步。

再讲一个特别有意思的特性叫 PWM dithering，也就是抖动技术。当电调工作在 96 千赫兹的 PWM 频率时，硬件只能提供 9 位的分辨率，也就是 512 级。但 Bluejay 通过逐帧交替使用两个相邻的 PWM 值，时间平均之后等效达到了 11 位的分辨率，也就是 2048 级。这个思路跟音频领域里的 Delta-Sigma 调制非常相似，是典型地用时间换精度的嵌入式设计技巧。

关于配置系统，Bluejay 支持通过 DShot 命令直接修改电调的参数，比如换相时机、退磁补偿、启动功率这些。飞控可以通过数字协议来调整电调的行为，不需要专门的编程器。参数修改后会持久化到 Flash 中模拟的 EEPROM 区域。由于 EFM8BB 芯片本身没有独立的 EEPROM，这里用 Flash 的一块区域来存储配置数据，写入前要先擦除整个页面，还要考虑擦写寿命的限制。

如果大家想动手读源码，我推荐按这个顺序来。首先是 Common.inc 文件，不到两百行，是整个固件的骨架。你能看到 MCU 选择逻辑、二十六种 PCB 布局的引脚定义、中断向量表、PCA 初始化宏和时钟切换宏。读完它可以对整个固件怎么组织有一个全景认识。然后是 Bluejay.asm 的前两百行，主要是寄存器定义、标志位和 EEPROM 参数表。这里特别值得关注的是每个位的注释，工程师在每个标志位后面都标注了它在什么条件下被使用。接下来重点读 Timer1 的中断处理程序，这是 DShot 解码的核心。最后是 BLHeliBootLoad.inc 和 tools/efm8load.py，帮你理解固件是怎么烧录进去的。

今天给大家准备了十个嵌入式概念卡片。第一个是反电动势，就是电机旋转时未通电的绕组切割磁感线产生的电压信号，无传感器电机控制全靠它来判断换相时机。难点是低速时反电动势信号极其微弱，必须先把电机转到一定速度才能可靠检测。第二个是六步换相，无刷电机每转一圈需要六次通电状态切换，时机偏差几度电角度就会导致剧烈振动。第三个是死区时间，同一桥臂的上下 MOSFET 切换时强制都关断的短暂时间，防止直通短路，太短会烧管子，太长会增加失真。第四个是可编程计数器阵列 PCA，它是独立于 CPU 的硬件 PWM 发生器，设置好参数后自动输出波形。第五个是 DShot 协议，飞控和电调之间的数字通信标准，比传统的模拟 PWM 信号更精确抗干扰。第六个是 GCR 编码，用于 DShot 遥测回传的数据编码方式。第七个是 Flash 模拟 EEPROM，在没有独立 EEPROM 的芯片上用 Flash 存配置数据，但注意 Flash 擦写寿命只有一万次左右。第八个是看门狗定时器，软件必须定期喂狗，否则硬件自动复位系统。第九个是模拟比较器，它硬件比较两个电压并输出数字信号，在 BEMF 过零检测中很关键。第十个是寄存器组切换，8051 架构的独有特性，每个中断可以有自己的寄存器组，减少压栈开销。

今天的动手任务不需要硬件。你可以用 Python 写一个 DShot 帧的编码解码模拟器。比如设定一个油门值 1050，组装成十六位帧，加上四位的 XOR 校验和，然后做反码处理模拟真实传输。再写解码部分验证校验和是否正确。这个小练习能让你透彻理解 DShot 协议的编解码原理。

我要问问大家三个工程判断问题。第一个问题是如果高油门下电机间歇性失步，示波器观察 BEMF 信号正常但退磁指标持续上升，你会先调退磁补偿还是换相时机，为什么？第二个问题是在只有 256 字节 RAM 的条件下要加一个 64 字节的环形日志缓冲区，你从寄存器组、堆栈、间接寻址区这些区域里牺牲哪一块，每个选择的影响是什么。第三个问题从软件工程师的角度对比通过 DShot 命令配置和通过 PC 配置器配置两种方式在可靠性、便捷性和可调试性上的差异，如果设计车载 OTA 方案会选哪种。

今天的复盘问题来自我对这个项目最深的感受：如果把 Bluejay 的 DShot 解码部分用 C 语言重写，你觉得最终代码会比汇编快还是慢，ROM 用量会变大还是变小？你的直觉和实际测量之间可能相差在哪里？

从工程的角度点评这个项目。可维护性方面，宏定义清晰，注释详尽，常量集中管理，但全汇编导致代码复用困难，不同布局文件之间的差异也难以 diff。可移植性方面深度绑定 EFM8BB 的外设寄存器，移植到其他平台等于重写。可靠性是强项，有多层故障检测机制，启动时序有完整补偿。可测试性比较弱，没有单元测试，依赖社区飞行测试。量产和运维方面做得很好，有上千种预编译的 HEX 文件，用户不需要自己搭建编译环境。

明天如果想继续延伸学习，可以看三个方向。一是对比 DShot 和 CANopen 协议的适用场景，为什么消费级无人机选择单线 DShot 而工业机器人用 CAN 总线。二是学习 BLHeli_32，看看当 RAM 从 256 字节变成 64KB 时架构设计会有哪些根本变化。三是结合之前讲过的 SimpleFOC，通过实际实验对比方波控制和 FOC 控制的效率差异。
