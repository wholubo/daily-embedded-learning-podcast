## 本周嵌入式实践项目：**串口日志系统 + Ring Buffer 最小实现**

### 1. 本周实践项目名称

**“非阻塞串口日志系统：用 Ring Buffer 实现嵌入式 printf 后端”**

你本周末要做一个很小但真实的嵌入式基础组件：

> 应用代码调用 `log_info("temp=%d", value)`，日志不会直接阻塞发送，而是先写入环形缓冲区，再由“串口发送任务/中断模拟器”慢慢取出发送。

---

### 2. 项目目标

实现一个最小可用的嵌入式日志模块，包含：

- 日志等级：`DEBUG / INFO / WARN / ERROR`
- 格式化输出接口，例如：
  - `log_info("boot ok")`
  - `log_error("sensor init failed: %d", err)`
- 内部使用 **Ring Buffer** 保存待发送数据
- 支持非阻塞写入：
  - 缓冲区满时丢弃新日志，或丢弃旧日志
  - 记录丢失日志数量
- 提供一个“串口发送后端”：
  - 有硬件时对接 UART
  - 无硬件时用桌面程序模拟 UART 慢速发送

---

### 3. 为什么这个项目对软件工程师转嵌入式有价值

软件工程师常见的日志系统通常默认：

- 磁盘/终端输出比较快
- 内存相对充裕
- 线程、锁、队列、标准库都可用
- `printf` 阻塞一下问题不大

但嵌入式里情况不同：

- UART 很慢，例如 115200 bps，每秒实际只有约 10KB 级别
- 中断里不能随便 `printf`
- RAM 可能只有几十 KB
- 日志过多会影响实时性
- 低优先级日志不能拖垮控制逻辑
- 缓冲区满了必须有明确策略

这个项目能让你从“会写 C/C++ 程序”进入到“理解嵌入式资源约束和实时性”的状态。

### 5. 软件工程类比与局限

#### 类比

可以把这个项目类比为后端服务里的异步日志队列：

```text
业务代码 -> 日志 API -> 内存队列 -> 后台线程 -> 文件/网络
```

嵌入式中类似：

```text
业务代码 -> log_xxx() -> ring buffer -> UART TX 中断/任务 -> 串口终端
```

#### 局限

但嵌入式和后端服务不同：

- 没有无限大的内存队列
- 不能随便动态分配内存
- 发送端速度极慢
- 中断上下文限制很多
- 日志系统本身不能破坏实时性
- 丢日志有时是可接受的，卡死系统不可接受

所以目标不是“日志一条不丢”，而是：

> 在资源有限的情况下，让日志尽量可用，同时不影响主系统运行。

```c
void rb_init(ring_buffer_t *rb, unsigned char *buf, unsigned int size);
```

```c
// elog.h
typedef enum {
    LOG_DEBUG,
    LOG_INFO,
    LOG_WARN,
    LOG_ERROR

void log_init(void);

void log_printf(log_level_t level, const char *fmt, ...);
void log_flush_step(void);

#define log_debug(...) log_printf(LOG_DEBUG, __VA_ARGS__)

```

- 日志格式类似：


- 日志过长时截断

- `log_flush_step()` 每次只发送少量字节，模拟 UART 慢速发送

```c
void log_flush_step(void) {
    unsigned char ch;
    for (int i = 0; i < 8; i++) {
        if (rb_read(&g_log_rb, &ch, 1) == 1) {
            uart_send_byte(ch);
        } else {
            break;
        }
    }
}
```

  - 简单方案：整条日志丢弃
  - `dropped_count++`

```text
[WARN] log dropped: 3
```

不强制做得很完美，重点是明确策略。

#### 下午：做“UART 慢速发送”模拟

写一个桌面模拟程序：

```c
int main(void) {
    log_init();
    log_set_level(LOG_DEBUG);

    for (int i = 0; i < 50; i++) {
        log_info("sensor value=%d", i);

        // 模拟主循环里每轮只发一点点
        log_flush_step();
    }

    // 最后 drain 剩余日志
    for (int i = 0; i < 1000; i++) {
        log_flush_step();
    }

    return 0;
}
```

观察：

- 缓冲区够大时是否完整输出
- 缓冲区很小时是否出现丢弃
- `log_flush_step()` 慢时是否触发丢日志
- 日志等级是否生效
